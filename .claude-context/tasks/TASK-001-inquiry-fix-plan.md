# Task #1: Fix Inquiry Table -- Create Missing Tables in Turso DB

**Date:** 2026-06-27
**Status:** Plan ready for implementation

---

## Problem

Submitting the inquiry form on `/offer` page returns "Internal error". The root cause is:
```
SQLITE_ERROR: no such table: main.inquiries
```

The Prisma schema (`prisma/schema.prisma:1001-1051`) defines two models:
- `Inquiry` mapped to `@@map("inquiries")` (lines 1001-1029)
- `InquiryItem` mapped to `@@map("inquiry_items")` (lines 1035-1051)

These tables were never created in the Turso production database. `prisma db push` does NOT work with Turso `libsql://` connection strings (confirmed in TASK-001-impl.md).

The API at `src/app/api/public/inquiry/route.ts` calls `prisma.inquiry.create()` which fails.

---

## Solution

Create a migration script (following the existing pattern in `scripts/create-reviews-table.ts`) that uses `@libsql/client` directly to execute raw `CREATE TABLE` SQL against Turso.

---

## Implementation Steps

### Step 1: Create migration script

**File:** `scripts/create-inquiry-tables.ts` (NEW)

Based on the Prisma schema models and the pattern from `scripts/create-reviews-table.ts`:

```typescript
import "dotenv/config";
import { createClient } from "@libsql/client";

const client = createClient({
  url: process.env.TURSO_DATABASE_URL!,
  authToken: process.env.TURSO_AUTH_TOKEN!,
});

async function main() {
  // Create inquiries table
  await client.execute(`
    CREATE TABLE IF NOT EXISTS inquiries (
      id TEXT PRIMARY KEY,
      name TEXT NOT NULL,
      email TEXT NOT NULL,
      phone TEXT,
      salonName TEXT,
      message TEXT,
      locale TEXT NOT NULL DEFAULT 'cs',
      status TEXT NOT NULL DEFAULT 'NEW',
      assignedTo TEXT,
      assignedAt DATETIME,
      internalNote TEXT,
      contactedAt DATETIME,
      completedAt DATETIME,
      createdAt DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
      updatedAt DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
    )
  `);

  // Create indexes on inquiries
  await client.execute(`CREATE INDEX IF NOT EXISTS idx_inquiries_status ON inquiries(status)`);
  await client.execute(`CREATE INDEX IF NOT EXISTS idx_inquiries_createdAt ON inquiries(createdAt)`);
  await client.execute(`CREATE INDEX IF NOT EXISTS idx_inquiries_email ON inquiries(email)`);

  // Create inquiry_items table
  await client.execute(`
    CREATE TABLE IF NOT EXISTS inquiry_items (
      id TEXT PRIMARY KEY,
      inquiryId TEXT NOT NULL,
      productId TEXT NOT NULL,
      productName TEXT NOT NULL,
      lengthCm INTEGER NOT NULL,
      color TEXT NOT NULL,
      quantity INTEGER NOT NULL,
      unit TEXT NOT NULL DEFAULT 'g',
      createdAt DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
      FOREIGN KEY (inquiryId) REFERENCES inquiries(id) ON DELETE CASCADE
    )
  `);

  // Create index on inquiry_items
  await client.execute(`CREATE INDEX IF NOT EXISTS idx_inquiry_items_inquiryId ON inquiry_items(inquiryId)`);

  console.log("Inquiry tables created successfully!");
}

main().catch(console.error);
```

### Step 2: Run the script

```bash
npx tsx scripts/create-inquiry-tables.ts
```

### Step 3: Verify

After running, test the inquiry form on `/offer` to confirm it creates records without error.

## Key Notes

- SQLite uses `TEXT` for string fields and `INTEGER` for int/boolean
- `DateTime` maps to `DATETIME` in SQLite
- `@default(now())` maps to `DEFAULT CURRENT_TIMESTAMP`
- `onDelete: Cascade` maps to `FOREIGN KEY ... ON DELETE CASCADE`
- Enum values (InquiryStatus: NEW, CONTACTED, COMPLETED, CANCELLED) are stored as TEXT in SQLite
- The `@id @default(cuid())` means the `id` column gets a CUID generated by Prisma client, not by the DB
